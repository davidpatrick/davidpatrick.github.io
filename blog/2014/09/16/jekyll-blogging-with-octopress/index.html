What Octopress give us

Octopress gives us a lot of bloat… There are a lot of preinstalled plugins in Octopress that I am not going to use, but I will rip them out later. You will also see our folder structure is very heavy compared to when we used Poole. However it affords us some advantages that I am going to utilize and that is better looking themes and decent plugins.

Deploying Octopress Generated Content

Unfortunately Deploying with Octopress is not as easy as it was with Poole. Since Octopress is a local Ruby Blogging Engine, we aren’t going to want to commit all the code in our Octopress folder. We actually need to set up our repo in a rake command rake setup_github_pages

This will ask us for our github pages repo and initialize that repo into /_deploy. You can go there to resolve any merge conflicts you might have with our new engines generated content.

After you have finished that let’s go ahead and deploy the code to Github using Octopress rake tasks.

This will generate our html and commit and push our code to the repo for us.

Final Conclusions

Octopress give us a lot of free stuff for a relatively cheap investment. However, even after all it gives us, it adds a layer of complexity that feels dirty when comparing to something as simple and elegant as Poole.

With Poole we are able to write all of our posts in markdown files and push the markdown files to our repo. This makes accessing the blog from different machines, or even remotely, A BREEZE! We can edit the markdown directly on github! It also had a simple beauty to it when you looked at the source on Github.

With Octopress we have to run rake tasks that convert our markdown pages to htmland then we have to run a deploy script to commit our changes. This also adds a complexity of setting up your Octopress engine on a second machine.

I will stick with Octopress for now on my personal blog because I enjoy the themes for it, but for the team blog, I will run with Poole and start customizing it to fit our needs.
